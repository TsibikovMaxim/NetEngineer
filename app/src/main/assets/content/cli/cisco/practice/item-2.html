Запись MAC-адреса в рабочую конфигурацию

Липкий MAC-адрес постоянно хранится в конфигурации запуска. Почему он называется липким? А всё потому, что его не нужно настраивать в ручную, нужно всего лишь разрешить функции port-security обнаружить MAC-адрес устройства обычным способом. И система IOS автоматически запишет этот MAC-адрес в рабочую конфигурацию.

Важный момент: чтобы поправить липкий MAC-адрес, нужно будет его предварительно удалить из конфигурации при помощи команды no switchport port-security mac-address sticky [mac], а затем добавить нужный MAC.

Switch>en // Привилегированный режим EXEC

Switch#conf t // Режим глобальной конфигурации

Switch(config)#int fa 0/1 // Выбор интерфейса

Switch(config-if)#switchport mode access // Сообщаем системе, что это абонентский порт

Switch(config-if)#switchport port-security // Активируем функцию switchport port-security на интерфейсе fa 0/1

Switch(config-if)#switchport port-security mac-address sticky // Указываем устройству, что нужно запомнить только один (несколько) MAC-адресов
