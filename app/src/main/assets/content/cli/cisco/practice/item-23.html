Настройка статического маршрута IPv6

Статические маршруты для протокола IPv6 настраиваются при помощи команды глобальной конфигурации ipv6 route.

Статический маршрут IPv6 можно реализовать следующим образом:

  • стандартный статический маршрут IPv6;
  • статический маршрут IPv6 по умолчанию;
  • объединённый статический маршрут IPv6;
  • плавающий статический маршрут IPv6 (это статические маршруты, административная дистанция которых больше, чем административная дистанция других статических маршрутов или динамических маршрутов).

Синтаксис команды

ipv6 route ipv6-prefix/prefix-length {ipv6-address | exit-intf}

  • ipv6-prefix (адрес удалённой сети назначения, который необходимо добавить в таблицу маршрутизации);
  • prefix-length (длина префикса удалённой сети назначения,  который необходимо добавить в таблицу маршрутизации);
  • ipv6-address (IP-маршрутизатора следующего маршрутизатора);
  • exit-intf (интерфейс для передачи пакетов).

Важно: Для того чтобы маршрутизатор мог осуществлять пересылку пакетов для IPv6, необходимо активировать одноадресную маршрутизацию командой ipv6 unicast-routing.

Router>en // Переходим в привилегированный режим EXEC
Router#conf t // Переходим в глобальный режим конфигурации
Enter configuration commands, one per line. End with CNTL/Z.
Router(config)#ipv6 unicast-routing // Активируем одноадресную маршрутизацию
Router(config)#ipv6 route 2001:DB8:ACAD:2::/64 2001:DB8:ACAD:4::2 // Указываем статический маршрут для следующего перехода
